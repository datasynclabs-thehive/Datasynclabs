## Overview

DataSynclabs governance is designed to be:
- Lightweight initially
- Centralized for validation
- Decentralizable by design

---

## Current Phase: Steward Governance

- Single maintainer (Queen role)
- Contributors act as Workers
- Decisions validated centrally
- Execution distributed

---

## Decision Types

### Operational
- Documentation changes
- Simulations
- Diagrams
Approved via issues.

### Architectural
- Core logic changes
- Incentive rules
Require discussion + explicit approval.

### Governance
- Voting
- Authority delegation
Not active yet.

---

## Evolution Path

Phase 1: Manual validation  
Phase 2: Rule-based automation  
Phase 3: On-chain or cryptographic governance

Governance automation follows proven coordination logic.
